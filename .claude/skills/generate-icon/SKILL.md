# 3D Icon Generator

회사의 Figma 레퍼런스 아이콘 스타일을 분석하여, 입력된 단어에 맞는 3D 아이콘을 생성하고 Figma에 업로드합니다.

## Phase 1: Figma 레퍼런스 읽기

1. Figma Official MCP의 `get_screenshot` 도구로 레퍼런스 페이지의 아이콘 캡처
   - Figma URL: https://www.figma.com/design/pKdkjgZU1mbhgpeFcklnYh/test?node-id=41-2
   - 해당 노드의 스크린샷을 가져옴
2. `get_metadata` 도구로 아이콘 노드 정보 (크기, 구조 등) 조회
3. 캡처한 이미지를 `styles/references/`에 저장

## Phase 2: 스타일 분석 & 프롬프트 템플릿

**`styles/icon-style-profile.md`가 이미 존재하고 `--refresh` 옵션이 없으면 이 Phase를 스킵합니다.**

1. Phase 1에서 가져온 레퍼런스 이미지를 Claude Vision으로 분석
2. 다음 항목을 추출하여 `styles/icon-style-profile.md`에 저장:
   - **3D 렌더링 스타일**: 클레이, 글래시, 아이소메트릭, 리얼리스틱 등
   - **조명**: 방향, 강도, 그림자 특성
   - **색상**: 주요 팔레트, 그라데이션 패턴
   - **배경**: 투명/컬러/그림자
   - **형태**: 모서리 둥글기, 크기 비율, 패딩
   - **질감**: 매트, 글로시, 반투명 등
   - **구도**: 카메라 앵글, 원근감

3. 분석 결과를 바탕으로 이미지 생성용 프롬프트 템플릿을 작성하여 `styles/prompt-template.md`에 저장
   - 템플릿은 `{icon_subject}` placeholder를 포함
   - 5-Part 프롬프팅 프레임워크 사용:
     1. **Image Type**: 3D 렌더링 스타일 명시
     2. **Subject**: `{icon_subject}` — 생성할 아이콘의 주제
     3. **Environment**: 배경 및 공간 설정
     4. **Technical Specs**: 조명, 카메라, 해상도
     5. **Constraints**: 텍스트 없음, 워터마크 없음, 단일 아이콘

## Phase 3: 대상 리서치 (Subject Research)

**입력 단어가 무엇인지 정확히 파악한 뒤에 이미지를 생성해야 합니다. 리서치 품질이 결과물의 정확도를 결정합니다.**

### Step 3-1: 기본 검색 (최소 2회 이상)
1. **한국어 검색**: `"{입력 단어}" 뜻` 또는 `"{입력 단어}" 이란`
2. **영어/원어 검색**: 해당 단어의 영문명 또는 원래 언어로 재검색
   - 예: "버터떡" → "Shanghai butter mochi" / "黄油年糕"
   - 한국어 검색만으로는 시각적 특징이 부족할 수 있으므로 반드시 교차 검증

### Step 3-2: 실물 외형 상세 파악 (WebFetch 최소 2페이지)
1. **레시피/만들기 페이지** 방문 — 완성품 사진 설명, 재료, 틀 모양 등 구체적 정보
2. **리뷰/블로그 페이지** 방문 — 실제 외형 묘사, 단면 사진 설명 등
3. 다음 항목을 **구체적 사실 기반**으로 파악:
   - 어떤 **틀/도구**로 만드는지 (형태 결정에 핵심)
   - **완성품 사진**에서 보이는 실제 모양
   - 겉과 속의 **색상 차이**
   - **크기** (한 입 크기? 손바닥 크기?)
   - **표면 패턴** (매끈? 골? 균열?)

### Step 3-3: 리서치 결과 정리 & 셀프 검증

**⚠️ HALLUCINATION 방지 규칙: 리서치 결과 정리 시, 검색 결과에 명시적으로 나온 정보만 사용할 것. 출처가 없는 정보는 절대 추가하지 않는다.**

리서치 결과를 정리하되, 각 항목에 **출처(어느 페이지에서 확인했는지)**를 명기합니다:
- **정체**: 이것이 무엇인지 한 줄 설명 — [출처]
- **형태 특징**: 모양, 크기, 구조 — [출처]
- **색상 특징**: 외부 색상, 내부 색상 — [출처]
  - ⚠️ 검색 결과에서 색상이 명시되지 않았다면 "확인 불가"로 표기. 추측하지 않는다.
- **표면 특징**: 매끈/울퉁불퉁/패턴/균열/파우더 등 — [출처]
- **핵심 디테일**: 이것을 이것답게 만드는 결정적 시각 요소 — [출처]

> **셀프 검증 체크리스트**:
> 1. "내가 정리한 각 항목에 출처가 있는가?" → 없으면 해당 항목 삭제 또는 추가 리서치
> 2. "내가 추측으로 채운 내용이 있는가?" → 있으면 삭제하고 "확인 불가"로 표기
> 3. "식감 묘사를 외형 묘사로 오해한 부분이 있는가?" → 식감(쫀득, 바삭)과 시각적 외형은 다르다
> 4. "이 설명만으로 이 물건을 본 적 없는 사람이 정확히 그릴 수 있는가?" → 아니면 리서치 추가

> **확인 불가 항목 처리**: 색상 등 확인 불가 항목은 프롬프트에서 해당 부분을 구체적으로 지정하지 않고, AI 모델이 일반적 지식으로 처리하도록 비워둔다. 잘못된 정보를 넣는 것보다 비워두는 것이 낫다.

## Phase 4: 아이콘 이미지 생성

1. `styles/prompt-template.md`에서 프롬프트 템플릿을 읽기
2. `{icon_subject}`를 Phase 3에서 파악한 **상세 설명**으로 치환
   - 단, 리서치한 실물 묘사를 그대로 넣지 말 것
   - 실물의 **핵심 형태와 구조**만 반영하고, 표현 방식은 클레이 모핑 스타일로 변환
   - 예: "코코아 파우더가 뿌려진 사실적 질감" → "코코아색 매트한 둥근 공 모양"
3. `mcp-image` MCP 서버의 `generate_image` 도구를 호출:
   - prompt: 치환된 프롬프트 (리서치 기반이되, 클레이 스타일로 변환된 묘사)
   - quality: "quality" (최고 품질)
   - aspect_ratio: "1:1" (아이콘이므로 정사각형)
4. 생성된 이미지를 `generated/{단어}.png`에 저장
5. 생성된 이미지를 `styles/references/` 폴더의 레퍼런스 아이콘과 **반드시 비교 검증**:
   - 레퍼런스 이미지 1~2개를 다시 읽어서 나란히 비교
   - 체크 항목:
     - [ ] 클레이/플라스틱 장난감 느낌인가? (사실적이면 실패)
     - [ ] 색상이 선명한 파스텔 톤인가? (탁하거나 사실적이면 실패)
     - [ ] 표면이 매끄럽고 단순화되어 있는가? (디테일 과다면 실패)
     - [ ] 배경이 순백(#FFFFFF)인가?
     - [ ] 바닥 드롭 섀도우가 없는가?
     - [ ] 앱 아이콘 프레임/둥근 사각형 컨테이너가 없는가? (오브젝트만 있어야 함)
   - 스타일 불일치 시: 프롬프트에 "more simplified, more clay-like, less realistic" 등을 보강하여 **1회 재생성**

## Phase 5: Figma 업로드

1. Figma Official MCP의 `use_figma` 도구를 사용:
   - 대상 파일: https://www.figma.com/design/pKdkjgZU1mbhgpeFcklnYh/test
   - "Generated Icons" 페이지가 없으면 새로 생성
   - 생성된 PNG 이미지를 프레임에 배치
   - 프레임 이름을 `$ARGUMENTS`로 설정
2. 업로드 완료 후 사용자에게 결과 보고:
   - 생성된 아이콘 로컬 경로
   - Figma 업로드 위치

## 오류 처리

- Figma OAuth 미인증 시: 브라우저 인증을 안내
- mcp-image 연결 실패 시: GEMINI_API_KEY 설정 확인 안내
- 이미지 생성 실패 시: 프롬프트를 조정하여 1회 재시도
- Figma 업로드 실패 시: 로컬 파일 경로를 안내하고 수동 업로드 방법 제시
