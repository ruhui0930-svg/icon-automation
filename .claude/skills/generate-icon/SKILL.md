# 3D Icon Prompt Generator

회사의 Figma 레퍼런스 아이콘 스타일을 분석하고, 입력된 단어에 맞는 **이미지 생성 프롬프트**를 만들어 사용자에게 전달합니다.
이미지 생성은 사용자가 직접 Gemini(nano banana)에 입력합니다.

## Phase 1: Figma 레퍼런스 읽기

1. Figma Official MCP의 `get_screenshot` 도구로 레퍼런스 페이지의 아이콘 캡처
   - Figma URL: https://www.figma.com/design/pKdkjgZU1mbhgpeFcklnYh/test?node-id=41-2
2. 캡처한 이미지를 `styles/references/`에 저장

## Phase 2: 스타일 분석 & 프롬프트 템플릿

**`styles/icon-style-profile.md`가 이미 존재하고 `--refresh` 옵션이 없으면 이 Phase를 스킵합니다.**

1. Phase 1의 레퍼런스 이미지를 Claude Vision으로 분석
2. 다음 항목을 추출하여 `styles/icon-style-profile.md`에 저장:
   - **3D 렌더링 스타일**: 클레이, 글래시, 아이소메트릭, 리얼리스틱 등
   - **조명**: 방향, 강도, 그림자 특성
   - **색상**: 주요 팔레트, 그라데이션 패턴
   - **배경**: 투명/컬러/그림자
   - **형태**: 모서리 둥글기, 크기 비율, 패딩
   - **질감**: 매트, 글로시, 반투명 등
   - **구도**: 카메라 앵글, 원근감
3. 분석 결과를 바탕으로 이미지 생성용 프롬프트 템플릿을 `styles/prompt-template.md`에 저장
   - 템플릿은 `{icon_subject}` placeholder 포함
   - 5-Part 프롬프팅 프레임워크 사용:
     1. **Image Type**: 3D 렌더링 스타일 명시
     2. **Subject**: `{icon_subject}` — 생성할 아이콘의 주제
     3. **Environment**: 배경 및 공간 설정
     4. **Technical Specs**: 조명, 카메라, 해상도
     5. **Constraints**: 텍스트 없음, 워터마크 없음, 단일 아이콘

## Phase 3: 대상 리서치 (Subject Research)

**입력 단어가 무엇인지 정확히 파악한 뒤에 프롬프트를 작성해야 합니다. 리서치 품질이 결과물의 정확도를 결정합니다.**

### Step 3-1: 기본 검색 (최소 2회 이상)
1. **한국어 검색**: `"{입력 단어}" 뜻` 또는 `"{입력 단어}" 이란`
2. **영어/원어 검색**: 해당 단어의 영문명 또는 원래 언어로 재검색
   - 예: "버터떡" → "Shanghai butter mochi" / "黄油年糕"

### Step 3-2: 실물 외형 상세 파악 (WebFetch 최소 2페이지)
1. **레시피/만들기 페이지** 방문 — 완성품 사진 설명, 재료, 틀 모양 등 구체적 정보
2. **리뷰/블로그 페이지** 방문 — 실제 외형 묘사, 단면 사진 설명 등
3. 파악할 항목:
   - 어떤 **틀/도구**로 만드는지 (형태 결정에 핵심)
   - **완성품 사진**에서 보이는 실제 모양
   - 겉과 속의 **색상 차이**
   - **크기**, **표면 패턴** (매끈? 골? 균열?)

### Step 3-3: 리서치 결과 정리 & 셀프 검증

**⚠️ HALLUCINATION 방지 규칙: 검색 결과에 명시적으로 나온 정보만 사용. 출처 없는 정보는 절대 추가 금지.**

각 항목에 **출처**를 명기:
- **정체**: 이것이 무엇인지 한 줄 설명 — [출처]
- **형태 특징**: 모양, 크기, 구조 — [출처]
- **색상 특징**: 외부/내부 색상 — [출처] (확인 불가 시 "확인 불가" 표기)
- **표면 특징**: 매끈/울퉁불퉁/패턴 등 — [출처]
- **핵심 디테일**: 결정적 시각 요소 — [출처]

> **셀프 검증**: 각 항목 출처 확인 → 추측 내용 삭제 → 식감≠외형 구분 → 본 적 없는 사람도 그릴 수 있는가?

## Phase 4: 최종 프롬프트 생성 & 전달

1. `styles/prompt-template.md`의 템플릿을 읽기
2. `{icon_subject}`를 Phase 3 리서치 기반 **클레이 스타일 묘사**로 치환
   - 실물 묘사를 그대로 넣지 말 것 → 핵심 형태/구조만 클레이 스타일로 변환
   - 예: "코코아 파우더가 뿌려진 사실적 질감" → "코코아색 매트한 둥근 공 모양"
3. 최종 프롬프트를 `generated/{단어}_prompt.txt`에 저장
4. 사용자에게 다음 형식으로 전달:

---
**📋 Gemini에 붙여넣을 프롬프트:**

```
{최종 프롬프트 전문}
```

**사용 방법:** Gemini(nano banana)를 열고 위 프롬프트를 그대로 붙여넣기 하세요.
생성 후 이미지를 `generated/{단어}.png`로 저장하면 Figma 업로드도 가능합니다.
---

## Phase 5: Figma 업로드 (선택 — 사용자가 이미지 생성 후 요청 시)

사용자가 이미지를 `generated/{단어}.png`에 저장한 뒤 업로드를 요청하면:

1. Figma Official MCP의 `use_figma` 도구를 사용:
   - 대상 파일: https://www.figma.com/design/pKdkjgZU1mbhgpeFcklnYh/test
   - "Generated Icons" 페이지가 없으면 새로 생성
   - 생성된 PNG 이미지를 프레임에 배치
   - 프레임 이름을 `{단어}`로 설정

## 오류 처리

- Figma OAuth 미인증 시: 브라우저 인증을 안내
- 프롬프트 생성 실패 시: 스타일 프로파일 재분석 후 재시도 (`--refresh` 옵션)
